# AdMob app-ads.txt 문제 해결 가이드

## ❌ "세부정보가 AdMob 계정의 정보와 일치하지 않는 것 같습니다" 오류

### 원인 분석

이 오류는 주로 다음 이유로 발생합니다:

1. **Publisher ID 불일치** (가장 흔한 원인)
   - `app-ads.txt` 파일의 Publisher ID가 AdMob 계정의 실제 ID와 다름
   
2. **파일 형식 문제**
   - 공백, 줄바꿈, 인코딩 문제
   - 불필요한 빈 줄

3. **파일 접근 문제**
   - 파일이 제대로 배포되지 않음
   - 웹사이트에서 접근 불가

### 해결 방법

#### 1단계: AdMob 계정에서 실제 Publisher ID 확인

**방법 1: 계정 설정에서 확인**
1. [AdMob 대시보드](https://apps.admob.com) 로그인
2. 왼쪽 메뉴에서 **설정** (⚙️ 아이콘) 클릭
3. **계정 정보** 선택
4. **Publisher ID** 확인 (형식: `pub-XXXXXXXXXXXXXXX`)

**방법 2: 앱 설정에서 확인**
1. AdMob 대시보드 → **앱** 섹션
2. 앱 선택 → **앱 설정** 클릭
3. **앱 정보** 탭에서 Publisher ID 확인

**방법 3: 광고 단위에서 확인**
1. AdMob 대시보드 → **앱** 섹션
2. 앱 선택 → **광고 단위** 클릭
3. 광고 단위 ID에서 Publisher ID 확인 (앞부분이 Publisher ID)

#### 2단계: app-ads.txt 파일 수정

**현재 파일 내용:**
```
google.com, pub-1444459980078427, DIRECT, f08c47fec0942fa0
```

**수정 방법:**
1. 1단계에서 확인한 **실제 Publisher ID**로 변경
2. 파일 형식 정확히 확인:
   - 각 필드는 쉼표와 공백(`, `)으로 구분
   - 줄 끝에 불필요한 공백 없음
   - 파일 끝에 빈 줄 없음 (또는 하나만)
   - UTF-8 인코딩 사용

**올바른 형식 예시:**
```
google.com, pub-[실제PublisherID], DIRECT, f08c47fec0942fa0
```

#### 3단계: 파일 재배포

1. 수정된 파일을 웹사이트 루트에 배포
2. 브라우저에서 `https://ambro.space/app-ads.txt` 접속하여 확인
3. 파일 내용이 정확한지 확인

#### 4단계: AdMob에서 재확인

1. AdMob 대시보드 → **앱** 섹션
2. 인증 대기 중인 앱 선택
3. **업데이트 확인** 버튼 클릭
4. 상태 확인 (최대 24시간 소요 가능)

### 체크리스트

- [ ] AdMob 계정에서 실제 Publisher ID 확인 완료
- [ ] `app-ads.txt` 파일의 Publisher ID를 실제 ID로 수정
- [ ] 파일 형식 확인 (공백, 줄바꿈, 인코딩)
- [ ] 파일이 `https://ambro.space/app-ads.txt`에서 접근 가능
- [ ] AdMob에서 "업데이트 확인" 버튼 클릭
- [ ] 24시간 대기 후 상태 재확인

### 추가 참고사항

- Publisher ID는 항상 `pub-`로 시작합니다
- Publisher ID는 계정당 하나이며, 모든 앱에서 동일합니다
- 파일 수정 후 즉시 반영되지 않을 수 있으므로 최대 24시간 대기 필요
- 여전히 문제가 발생하면 AdMob 고객 지원에 문의하세요
